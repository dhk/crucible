# Metadata Schema

Crucible stores debate artifacts as structured files in the repository. This document defines the canonical schema for each artifact type.

## Concept registry (`concepts/registry/`)

| File | Contents |
|---|---|
| `concepts.csv` | id, name, agent, state, adoption, originCycle, group, desc |
| `edges.csv` | from, to, type (support/contradict/mutation/lineage), cycle |
| `mutations.csv` | conceptId, fromState, toState, cycle, rationale |

## Cycle artifacts (`reviews/cycles/`)

Each cycle produces a directory: `reviews/cycles/CYC-NNN-<agent>/`

```
document.md          the document at end of this cycle
rationale.md         why this agent made the changes it made
concept-delta.yaml   added/mutated/removed concept ids
pr-summary.md        generated PR body
```

## Observatory snapshot (`visualizations/observatory.json`)

Generated by `make build-observatory`. See `docs/architecture/lineage-storage.md` for the build pipeline.
